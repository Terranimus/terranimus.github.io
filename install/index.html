Installing TerranSoul

Two ways in: grab a prebuilt installer, or build it from source. Either way you end up in the same first-launch brain setup wizard.

Prebuilt installer

Download the build for your platform from GitHub Releases.

PlatformFormat
Windows.msi / .exe
macOS.dmg (Apple Silicon + Intel)
Linux.deb / .rpm / .AppImage

Run the installer, launch TerranSoul, and the first-launch wizard takes over from there — nothing else to configure by hand.

Build from source

If you'd rather build it yourself:

git clone https://github.com/TerranSyn/TerranSoulApp.git
cd TerranSoulApp
npm install
cargo tauri dev

Prerequisites for building from source:

PrerequisiteMin version
Node.js≥ 20
Ruststable
Tauri CLIlatest
WebView2any (Windows only, auto-detected)
Every supported AI coding agent — Copilot, Cursor, Claude Code, Codex, Roo Code, Windsurf, and TerranSoul's own coding subsystem — has a built-in /setup-prerequisites command that checks and installs whatever's missing for your OS, so you don't have to chase down toolchain versions by hand.

First launch: choosing a brain

Whichever install path you took, the app opens into a setup wizard that asks how you want to power the brain. There are three tiers:

  • Free cloud API — a free-tier provider such as OpenRouter, Gemini, NVIDIA NIM, or Pollinations, using your own key/token. Rate limits vary by provider.
  • Paid cloud API — OpenAI, Anthropic, or another paid provider with your own API key. Highest quality, needs a paid key.
  • Local LLM — runs entirely on your machine, no internet required once the model is downloaded. The wizard walks you through Ollama, LM Studio, llama.cpp (MTP), or driving a Claude CLI subscription you already have.

Local mode via Ollama is the path most people land on first: the wizard checks whether Ollama is installed, offers to install it if not, profiles your hardware, and pulls a recommended model. That download can take a few minutes depending on your connection and the model size — everything else is automatic once it finishes.
