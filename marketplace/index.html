External-agent plugin marketplace

A 223-entry catalog of agent plugins pulled from two other ecosystems, browsable and installable from the Market tab without leaving TerranSoul.

What it is

TerranSoul ships an embedded catalog (mcp-data/marketplace/external-agent-catalog.json) generated from the Hermes Agent plugin tree and the OpenClaw extension tree. Every entry is a validated plugin manifest — id, display name, kind, license, provenance — checked in as a diff, refreshed manually rather than synced live in the background. 223 is not a rounded marketing number: it's asserted by a unit test (hermes_count + openclaw_count == catalog.len()) so the catalog can't silently drift out of sync with its two sources.

Browsing

The Market tab calls marketplace_list_external_agent_plugins, which loads the catalog and cross-references each entry against PluginHost so you can see install state (not-installed / installed / quarantined, etc.) alongside each plugin's kind, license, and provenance.

Installing a plugin

Install isn't a single click-and-done download — it's a supervised, four-step flow, because an external-agent plugin is itself agent-authored code with real capabilities:

  1. Consent. You review and explicitly acknowledge a consent dialog before anything happens.
  2. Dispatch. marketplace_request_external_plugin_install generates an install brief and dispatches it to TerranSoul's own agentic-coding roster (see Agentic coding) as a tracked WorkflowPlan.
  3. Finalize. Once that workflow plan reaches Completed, marketplace_finalize_external_plugin_install registers the plugin in PluginHost and runs a post-install MCP connection check. Safe to call repeatedly — it reports a pending status until the plan is actually done.
  4. Uninstall (symmetric). marketplace_uninstall_external_plugin dispatches a cleanup brief to the same roster, then removes the plugin's registration.
This catalog is a superset, not a leaderboard — the same repository lets you browse Hermes-sourced and OpenClaw-sourced entries side by side rather than picking one ecosystem. There's no separate per-plugin doc page for each of the 223 entries here; that's a deliberate call (see below), not an oversight.

Why one hub page, not 223

Hermes documents its own skills with one auto-generated page per skill (174 of them). We looked at that pattern and decided it's the wrong shape for a catalog assembled from two other projects' plugin trees: the source-of-truth for what each plugin does already lives in Hermes's and OpenClaw's own docs, and mirroring 223 thin pages here would just be another thing to keep in sync as those upstream projects change. This page plus the in-app listing (which shows the full manifest per entry) covers the "what is this and how do I get it" question without duplicating content we don't own.
