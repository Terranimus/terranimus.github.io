Connecting chat channels

TerranSoul can talk to you outside the app window too — over Telegram, Discord, Slack, or WhatsApp. All four share one adapter contract; each has its own auth setup.

How it's built

Every channel implements the same ChannelAdapter trait: send() to push a message out, poll_inbound() to drain whatever's arrived. Bot tokens and other secrets go into the same AES-256-GCM credential vault the app's other connectors already use, keyed channel:<kind> — nothing is stored in plain text and nothing is committed to source. Per-channel toggles (enabled, parse mode, base-URL overrides) are brain-resident config rows, not hardcoded constants.

The four channels split into two shapes because the platforms themselves are architecturally different:

ChannelInbound modelOutbound
TelegramScheduler-driven HTTP long-pollREST sendMessage
DiscordPersistent Gateway WebSocketREST POST
SlackPersistent Socket Mode WebSocketREST POST
WhatsAppSidecar process (Baileys/WhatsApp Web protocol)Sidecar send command

Whatever comes in over any channel gets routed through the same session-routing pipeline the app's own chat UI uses, so a conversation started in Telegram sees the same memory and brain behind it as a conversation started in the app.

Commands you'll use

  • channel_save_credential(channel, json_str) — encrypt and store a channel's secret blob (bot token, app token, device label — whatever that platform needs).
  • channel_list() — see which channels are configured and enabled.
  • channel_send(channel, text, recipient?, parse_mode?) — send a message; omit channel to broadcast to every enabled+configured one.
There's no dedicated settings-tab UI for channels yet — you configure them by calling these commands directly (e.g. from the app's dev console, or by pointing an MCP-connected agent at them). If you're comfortable with that, the per-channel pages below have the exact JSON shape each one expects.

Pick a channel

Telegram →

Bot API token + chat id. Simplest to set up — no gateway to keep alive.

Discord →

Bot token, persistent Gateway connection, auto-reconnect with backoff.

Slack →

Bot token + App-Level token, Socket Mode (no public URL needed).

WhatsApp →

QR-code pairing via a Baileys sidecar process, no Business API account needed.
