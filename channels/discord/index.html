Discord

Discord's bot API is architecturally the opposite of Telegram's: instead of a scheduler-driven poll, a bot holds one persistent WebSocket connection ("the Gateway") open for its whole lifetime.

Status & capabilities

Bidirectional. A supervisor connects to the Gateway at host boot, identifies once, heartbeats on the interval Discord dictates, and receives message events the instant they happen — no polling delay. If the connection drops, the supervisor reconnects with backoff, resuming the session where possible instead of re-identifying from scratch. Outbound sends are a normal REST POST to {base}/channels/{id}/messages with Authorization: Bot <token>.

Auth setup

  1. Create an application at the Discord Developer Portal, add a Bot user, and copy its token.
  2. Under the bot's settings, enable the Message Content Intent (Discord requires this explicitly for a bot to read message text).
  3. Generate an OAuth2 invite URL with the bot scope and Send Messages/Read Message History permissions, and use it to add the bot to your server.
  4. Get the numeric id of the channel you want it to post in (right-click the channel → Copy Channel ID; Developer Mode must be on in Discord's settings).
  5. Save the credential:
channel_save_credential(
  channel: "discord",
  json_str: '{"token": "<bot token>", "chat_id": "<default channel id>"}'
)

Then flip channels.discord.enabled to true.

Config

KeyMeaning
channels.discord.enabledMust be true for the gateway supervisor to run.
channels.discord.gateway_urlOptional override of the live wss://gateway.discord.gg endpoint — mainly a test seam, not something you'll normally set.
channels.discord.base_urlOptional REST API base override.

Troubleshooting

  • Bot connects but never sees messages — check the Message Content Intent is enabled in the Developer Portal; without it, Discord silently strips message text from the events the Gateway sends.
  • Repeated reconnects — normal in small numbers (Discord Gateway connections do drop); persistent reconnect loops usually mean an invalid/revoked token.
  • "chat_id has no destination" — no default channel id saved and no recipient passed to channel_send.
