FAQ & troubleshooting

One page, pulled from the app's own tutorials rather than invented — if a fix here stops working, the underlying tutorial (linked at the bottom) has more detail.

Install & first launch

ProblemFix
Installer blocked on macOSRight-click → Open, or System Settings → Privacy & Security → Open Anyway (macOS blocks unsigned/unnotarized installers by default).
Missing libs on LinuxInstall: sudo apt install webkit2gtk-4.1 libgtk-3-0 libsoup-3.0-0 libayatana-appindicator3-1
Character/VRM not loadingVerify the file is a valid VRM 0.0 or 1.0 model; try a bundled model first to isolate whether the file itself is the problem.
Pet mode isn't transparentYour OS needs compositing support for transparent windows — usually fine on Windows/macOS, may need a compositor enabled on Linux.

Brain / local model setup

ProblemFix
No response from the brain at allSettings → Brain tab — confirm a provider is actually configured, not left on the setup wizard's default.
"LM Studio not running"Open LM Studio and start its local server before retrying.
No models listed (LM Studio)Download a model inside the LM Studio GUI first — TerranSoul only lists what's already local.
Embedding fails / no RAG sources in an answerAn embedding model isn't loaded — load one (e.g. qwen3-embedding-0.6b in LM Studio, or nomic-embed-text on Ollama) in the Market tab.
Responses are slowThe model is probably too large for your GPU to fully offload — try a smaller parameter count.
Connection refused to a local providerCheck the base URL matches what the provider is actually listening on (LM Studio defaults to http://127.0.0.1:1234).
Memory/Knowledge tab looks empty right after a chatMemory extraction runs on a turn cadence, not every single message — keep chatting, or trigger an extract manually if the UI offers it.

Channels

See the channel-specific troubleshooting tables on each page: Telegram, Discord, Slack, WhatsApp. The one thing common to all four: nothing sends or receives until channels.<kind>.enabled is true and a credential has actually been saved — enabling without credentials, or saving credentials without enabling, both look like "it's configured but does nothing."

MCP (connecting other AI agents)

ProblemFix
npm run mcp fails to buildRun npm install first; confirm Node.js ≥ 18.
Port 7423 already in useUsually fine — another MCP instance is already running and the start script will reuse it if it's healthy. Verify with curl http://127.0.0.1:7423/health.
Token rejectedRe-read the current token from mcp-data/mcp-token.txt — it's regenerated on each server start.
brain_search returns emptyCheck mcp-data/shared/memory-seed.sql exists and was actually loaded — an empty store returns empty results correctly, that's not a bug by itself.
Code-intelligence tools not availableThey require the code_read capability and an indexed repo — a plain brain-only connection won't see them.
Copilot / Claude Code / Codex / Cursor won't connectVerify the editor's own MCP config file has the right URL/token (.vscode/mcp.json, claude_desktop_config.json, ~/.codex/config.json, ~/.cursor/mcp.json respectively) and restart the editor after saving it.

Sharing a brain over LAN

SymptomCheck
"LAN mode not enabled"Enable LAN brain sharing/discovery in the Brain view before starting sharing.
"MCP server must be running"Start the MCP server (AI Coding Integrations) before starting sharing.
One device can't discover the otherConfirm both are on the same subnet and UDP 7424 isn't blocked by a firewall.
Manual connect failsDouble-check the peer's LAN IP, MCP port, and access mode — if token mode is selected, verify the token too.
Search returns nothing after connectingThe peer may not have ingested/tagged any documents yet, or the query may need to be more specific.
A browser/hosted client can't connectUse native desktop/mobile pairing — a public HTTPS page can't receive UDP discovery packets.

Miscellaneous

Can I run two local Ollama models at once?

Only if free RAM minus roughly a 1.5 GB OS reserve covers the sum of both models' sizes. Usually no on an 8 GB laptop, usually fine on 32 GB+.

What happens to an in-progress coding workflow if I force-quit the app?

Everything written to SQLite up to that point survives. On relaunch it shows as Resuming, but the child process itself was killed, so its in-memory state is gone — retry with a fresh workflow using the same prompt.

Can I add a coding CLI that isn't Claude/Codex/Gemini?

Yes — pick Custom and supply the binary name (alphanumerics plus -/_/. only). It must resolve via $PATH; absolute paths are rejected.

Does the agent roster / device configuration sync across my devices?

Not today — per-device JSON config files are local-only.
