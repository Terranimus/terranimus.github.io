Telegram

The simplest of the four — no gateway process to keep alive, just a bot token and a scheduled poll.

Status & capabilities

Bidirectional. Inbound messages arrive via a scheduler-driven HTTP long-poll (not a webhook); outbound sends are a plain REST call to sendMessage. Both directions run through the same session-routing pipeline every other channel uses, so a Telegram conversation gets the same memory/brain behind it as the in-app chat.

Auth setup

  1. Message @BotFather on Telegram, run /newbot, and follow the prompts to get a bot token (looks like 123456789:ABCdefGhIJKlmNoPQRstuVWXyz).
  2. Start a chat with your new bot (or add it to a group) and send it any message so Telegram will let it reply — then find your chat id, e.g. via the bot's getUpdates response or a helper bot like @userinfobot.
  3. Save the credential:
channel_save_credential(
  channel: "telegram",
  json_str: '{"token": "<bot token>", "chat_id": "<default chat id>"}'
)

Then flip channels.telegram.enabled to true in the brain-resident config. chat_id becomes the default recipient for channel_send when you don't pass one explicitly.

Config

KeyMeaning
channels.telegram.enabledMust be true for polling/sending to run. Defaults to false.
channels.telegram.parse_modeOptional — e.g. MarkdownV2, if you want formatted messages.
channels.telegram.base_urlOptional override, for pointing at a self-hosted Telegram Local Bot API Server instead of api.telegram.org.

Troubleshooting

  • Bot never replies — check channels.telegram.enabled is true and the token was saved without extra whitespace/quotes.
  • "chat_id has no destination" — you sent without a chat_id saved and without passing recipient to channel_send.
  • Group chat doesn't respond — bots need to be explicitly added to a group, and Telegram's privacy mode may hide messages the bot didn't @-mention; disable privacy mode in BotFather if you want it to see everything.
